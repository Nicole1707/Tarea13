Estructura de un compilador

1. Análisis léxico
Convierte el código fuente en una secuencia de tokens, que son las unidades básicas del lenguaje.
2. Análisis sintáctico
Organiza los tokens en una estructura de árbol, conocida como árbol sintáctico, para entender la estructura del código.
3. Análisis semántico
Verifica la coherencia de los datos y las operaciones en el código, asegurándose de que sigan las reglas semánticas del lenguaje.
4. Generación de código intermedio
Crea una representación intermedia del código, que facilita su optimización y la traducción a código de máquina.
5. Optimización de código
Mejora la eficiencia del código intermedio para que el código final sea más rápido y consuma menos recursos.
6. Generación de código
Traduce el código intermedio optimizado a código máquina específico para el sistema de destino.
7. Ensamblaje y enlazado
Convierte el código máquina en un formato ejecutable y lo enlaza con las bibliotecas necesarias para su ejecución.
